## Problèmes détectés

### 🔴 Bloquants

#### B1 — Test unitaire échoue : modèle Anthropic attendu `sonnet`, reçu `opus`
- **Catégorie** : tests
- **Fichier** : `apps/api/src/modules/integrations/llm/llm.service.spec.ts:167`
- **Description** : Le test `uses claude-sonnet model when history is provided` attend `toContain('sonnet')` mais le modèle a été changé en `claude-opus-4-6` dans les deux branches (avec et sans historique).
- **Impact utilisateur** : CI bloquée, aucun déploiement automatique fiable.
- **Correction proposée** : Mettre à jour l'assertion du test pour refléter `claude-opus-4-6`.
- **Estimation effort** : XS

#### B2 — 337 erreurs TypeScript dans l'API (spec files + seed + DTO)
- **Catégorie** : types
- **Fichiers** :
  - `src/database/seed.ts:749` — `level_order` n'existe pas encore (contrainte unique ajoutée au schema mais Prisma client pas régénéré)
  - `src/modules/ai/ai.dto.ts` — propriétés non initialisées dans les classes DTO
  - `src/modules/integrations/*/**.spec.ts` — `@types/jest` absent du tsconfig
- **Impact utilisateur** : Build Docker peut échouer si Railway régénère le client Prisma sans `prisma generate`.
- **Correction proposée** :
  1. Ajouter `"strictPropertyInitialization": false` dans `tsconfig.json` ou ajouter `!` aux propriétés DTO
  2. Ajouter `"types": ["jest"]` dans `tsconfig.json`
  3. Le `level_order` se résoudra après `prisma generate` dans le build
- **Estimation effort** : S

#### B3 — `/ai/feedback` et `/ai/translation-check` retournent 404 (AiModule pas déployé)
- **Catégorie** : build/déploiement
- **Fichier** : `apps/api/src/app.module.ts`
- **Description** : `AiModule` a été ajouté au code et pushé sur GitHub, mais Railway n'a pas encore redéployé avec ce nouveau code (image Docker cachée).
- **Impact utilisateur** : Les fonctionnalités IA (bouton "Explique-moi") ne fonctionnent pas.
- **Correction proposée** : Bumper le `cache-bust` dans le Dockerfile pour forcer un rebuild Railway.
- **Estimation effort** : XS

#### B4 — Tokens Supabase stockés dans `AsyncStorage` (non sécurisé)
- **Catégorie** : sécurité
- **Fichier** : `apps/mobile/src/services/supabase.ts`
- **Description** : Le client Supabase utilise `AsyncStorage` pour stocker les tokens JWT. Sur iOS/Android, `AsyncStorage` n'est pas chiffré. Les tokens d'accès sont lisibles si le device est compromis.
- **Impact utilisateur** : Risque de vol de session sur device jailbreaké/rooté.
- **Correction proposée** : Utiliser `expo-secure-store` via un adapter `LargeSecureStore` pour Supabase.
- **Estimation effort** : M
